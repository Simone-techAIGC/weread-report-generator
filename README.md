
# 📊 微信读书 WeRead 智能阅读报告生成器

> 一键生成个人微信读书数据可视化报告 + 阅读画像，帮你量化阅读习惯，发现阅读规律。

## 🎯 项目背景

作为重度微信读书用户，我一直希望了解自己的阅读行为——每周读了多少？喜欢什么类型？阅读节奏如何？但官方 App 只提供简单的阅读时长，缺少深度分析。

我用 **OpenCode + weread-skills** 构建了这套自动化报告系统：只需一条自然语言指令，AI 就会自动获取数据、生成交互式 HTML 报告，并基于历史数据给出个人阅读画像和改进建议。

**整个过程不需要写任何代码**，全部通过自然语言驱动。

---

## ✨ 功能特性

- 📊 **量化阅读统计**：总时长、阅读天数、日均时长、书籍排行、分类偏好、笔记/划线数
- 🧠 **个人阅读画像**：基于历史数据，分析阅读节奏、深度、广度，并给出可执行的成长建议
- 📈 **可视化报告**：使用 ECharts 5 生成交互式图表（条形图、饼图、折线图、雷达图）
- 🕒 **灵活的时间维度**：支持本周、本月、本年任意切换
- 📝 **完整字段展示**：书名、作者、出版社、ISBN 等全部完整显示，无截断
- ⚡ **一键生成**：在 OpenCode 中用自然语言触发，也可保存为自定义命令，重复使用

---

## 🛠 技术栈

- **[OpenCode](https://opencode.ai)**：AI 编程助手，负责理解指令、调用技能、生成 HTML 代码
- **[weread-skills](https://github.com/opencode-skills/weread-skills)**：OpenCode 技能，封装了微信读书 API 的调用逻辑，自动读取环境变量中的密钥
- **[ECharts 5](https://echarts.apache.org/)**：开源可视化图表库，通过 CDN 加载
- **HTML + CSS + JavaScript**：纯前端报告，无需部署，双击即可在浏览器中打开

## 📦 安装与配置

### 前置条件
- 已安装 [Node.js](https://nodejs.org/)（建议 v18 以上）
- 拥有微信读书 API 密钥（可从第三方服务或官方授权渠道获取）

### 步骤

#### 1. 安装 OpenCode（全局）
打开终端（Terminal），粘贴以下命令并回车：
```bash
npm install -g opencode
```

#### 2. 安装 weread-skills 技能
将以下内容发给 opencode(或者其他Agent)：
```bash
npx skills add Tencent/WeChatReading -g
```


#### 3. 配置微信读书 API 密钥（临时生效，仅当前终端窗口有效）
如果你只是想快速测试，可以在终端中执行：
```bash
export WEREAD_API_KEY="你的真实API密钥"
```
**注意**：关闭这个终端窗口后，密钥就会失效，下次需要重新输入。

---

#### 4. 配置微信读书 API 密钥（永久生效，推荐）
 详细步骤:
如果希望以后每次打开终端都能自动加载密钥，不用重复输入，请按照以下步骤操作。  
（即使你不懂编程，只要一步一步跟着做，就能完成。）

---

**第一步：确定你使用的是哪种 Shell（命令行环境）**
- 打开终端，输入以下命令并回车：
  ```bash
  echo $SHELL
  ```
- 如果输出结果包含 `/zsh`，说明你用 **Zsh**，配置文件是 `~/.zshrc`。
- 如果输出结果包含 `/bash`，说明你用 **Bash**，配置文件是 `~/.bashrc`。
- 如果你不确定，也可以两个文件都添加，不会有副作用。

---

**第二步：找到你的配置文件**
- 你的配置文件就在你的电脑用户文件夹里，文件名以点开头（比如 `.zshrc`），默认是隐藏的。
- **最简单的打开方式**：在终端中输入下面的命令，它会用文本编辑器打开该文件（我们用的是 `nano`，一个简单易用的编辑器）：
  - 如果你用的是 Zsh（上一步输出带 `zsh`），输入：
    ```bash
    nano ~/.zshrc
    ```
  - 如果你用的是 Bash（上一步输出带 `bash`），输入：
    ```bash
    nano ~/.bashrc
    ```

---

**第三步：在配置文件中添加密钥**
- 文件打开后，你会看到一些已有的内容（可能很多，也可能很少）。**不要删除任何原有的内容**。
- 用键盘的方向键将光标移到文件的**最后一行**，然后按回车键，新建一个空白行。
- 在空白行中**复制粘贴**下面这行内容（记得把 `你的真实API密钥` 替换成你自己的密钥）：
  ```
  export WEREAD_API_KEY="你的真实API密钥"
  ```
  例如，如果你的密钥是 `sk-abc123xyz`，就写成：
  ```
  export WEREAD_API_KEY="sk-abc123xyz"
  ```

---

**第四步：保存并退出**
- 按键盘上的 **`Ctrl + O`**（字母 O，不是数字0），这时会提示你确认文件名。
- 直接按 **回车键** 确认保存。
- 然后按 **`Ctrl + X`** 退出编辑器，回到终端界面。

---

**第五步：让配置立即生效**
- 在终端中执行以下命令（注意：如果上一步你修改的是 `.zshrc`，就执行第一条；如果是 `.bashrc`，就执行第二条。如果你不确定，两条都执行也没关系）：
  ```bash
  source ~/.zshrc
  ```
  或
  ```bash
  source ~/.bashrc
  ```
- 执行后，密钥就已经加载到当前终端环境中了。

---

**第六步：验证密钥是否配置成功（可选，推荐）**
- 输入以下命令并回车：
  ```bash
  echo $WEREAD_API_KEY
  ```
- 如果终端输出了你刚才填写的密钥内容（例如 `sk-abc123xyz`），说明配置成功！
- 如果输出是空白的，说明密钥没有正确加载，请检查上一步的 `source` 是否执行了，或者文件名是否正确。

---

> **提示**：以后每次打开新终端，系统都会自动加载这个密钥，你无需再手动输入。

---

#### 5. （可选）设置全局提示词
如果你希望 AI 默认按特定风格回复（例如始终使用中文、不输出推理过程等），可以创建 `~/.opencode/instructions.md` 文件，并在 `~/.config/opencode/opencode.json` 中引用它：
```json
{
  "instructions": ["~/.opencode/instructions.md"]
}
```

## 🚀 使用方法

### 方式一：直接对话（最灵活）
在终端运行：
```bash
opencode run "请利用 weread-skills 生成我本周的微信读书报告，包含统计数据和阅读画像"
```
系统会自动调用技能、获取数据、生成 HTML 报告并保存到当前目录。

### 方式二：自定义命令（最快捷）
将完整的提示词（包含数据获取、图表生成、画像分析等所有要求）保存为 `~/.opencode/commands/report.md`，然后在 OpenCode TUI（终端界面）中输入 `/report` 即可一键生成。

## 📄 生成报告示例

执行后会在当前目录生成一个独立的 HTML 文件（如 `weread_full_report_with_profile.html`），双击即可在浏览器中打开。

报告包含以下内容：

- 📊 **顶部概览卡片**：总阅读时长、阅读天数、日均时长、总书籍数、总笔记数
- 📈 **水平条形图**：书籍阅读时长排行（按降序，完整显示书名和作者）
- 🥧 **环形图**：各书籍阅读时长占比（显示完整书名和百分比）
- 📉 **每日趋势折线图**（如有数据）：展示本周每日阅读时长变化
- 📊 **分类偏好柱状图**（如有数据）：展示常读分类分布
- 🧠 **个人阅读画像**（文字+雷达图）：
  - 阅读节奏：规律性、日均时长、阅读天数占比
  - 阅读偏好：最爱作者、最常分类、平均每本书阅读周期
  - 阅读深度：笔记/划线频率、完成率、深度指数
  - 成长建议：3~5 条可执行的具体改进建议
- 📋 **完整数据表格**：书籍列表、作者、出版社、ISBN、阅读时长、进度

所有图表支持鼠标悬浮查看详情，图例可点击筛选，页面适配电脑和手机屏幕。

## 🙋 常见问题

**Q: 需要编程基础吗？**  
A: 不需要。整个流程通过自然语言与 OpenCode 对话完成，只需按照安装步骤配置技能和环境变量即可。

**Q: 如何获取微信读书 API 密钥？**  
A: 微信读书官方未公开提供 API，目前密钥可通过第三方服务（如“微信读书助手”等）或合作渠道获取。请自行查找可信来源。

**Q: 报告可以自定义吗？**  
A: 可以。你只需要修改提示词中的描述，比如调整图表类型、增加或删减分析维度、改变配色等。AI 会根据你的新指令重新生成。

**Q: 生成的 HTML 报告可以分享吗？**  
A: 可以。HTML 文件是独立的，包含所有样式和脚本（ECharts 通过 CDN 加载），可以直接发送给他人，或部署到静态托管服务。

**Q: 为什么某些图表显示“暂无数据”？**  
A: 这可能是因为微信读书 API 未返回对应字段（如每日趋势、时段分布）。报告会友好地显示占位信息，不影响其他部分。

**Q: 我按步骤配置了环境变量，但运行时报错说找不到密钥，怎么办？**  
A: 请先执行 `echo $WEREAD_API_KEY` 检查是否输出正确密钥。如果没有输出，请重新执行 `source ~/.zshrc`（或 `~/.bashrc`），并确保文件中的 export 语句拼写正确，且密钥值没有多余的空格。
