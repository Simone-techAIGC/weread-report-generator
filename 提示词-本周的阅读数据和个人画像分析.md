请利用 weread-skills 获取微信读书数据，生成一份包含阅读统计报告和个人阅读画像的交互式 HTML 可视化报告。

## 一、数据获取（完整提取，不截断）
请依次调用以下 API，并**完整提取**所有字段：

1. `/readdata/detail?mode=weekly`（默认本周，用户可指定 month/year）
   - 提取每本书：`title`（完整书名）、`author`（完整作者，若为数组用「、」连接所有作者）、`readingTime`（阅读分钟数）、`progress`（阅读进度百分比）、`cover`（封面 URL 若有）
   - 同时提取 `readDays`（阅读天数）、`dailyReadTime`（每日阅读时长，若存在）

2. `/readdata/stat?mode=weekly`
   - 提取：`totalReadingTime`（总时长）、`totalNotes`（总笔记数）、`totalBookmarks`（总划线数）

3. 对阅读时长最长的前 5 本书，调用 `/book/info`
   - 必须提取：`title`、`author`（完整拼接）、`publisher`（完整出版社名称）、`isbn`（ISBN 编号，若有）、`category`（分类）、`wordCount`（字数）

4. （可选）`/shelf/sync` 获取书架分类，用于分析阅读偏好。

5. **缺失数据处理**：若某接口无数据（如时段分布、每日趋势），直接标记“暂无数据”，不中断生成。

## 二、字段展示规则（修复截断）
- 所有文本字段（书名、作者、出版社、ISBN）必须在 HTML 中**完整显示**，不得截断或只取首字符。
- 作者若为数组，完整拼接（如“张三、李四”）。
- HTML 表格和卡片中设置 `white-space: normal; word-break: break-word;`，确保长文本自动换行。
- 任何缺失字段显示“暂无数据”，不得留空或报错。

## 三、阅读统计报告（量化指标 + 图表）
页面顶部展示概览卡片：
- 总阅读时长（分钟）、阅读天数、日均时长、总书籍数、总笔记数、总划线数

图表部分（使用 ECharts 5，CDN 加载）：
1. **水平条形图**：书籍阅读时长排行（降序），显示完整书名和作者
2. **环形图**：各书籍阅读时长占比，显示完整书名和百分比
3. **折线图**（若有每日数据）：展示本周每日阅读时长变化
4. **分类柱状图**（若有分类数据）：展示常读分类分布
5. 所有图表支持悬浮显示详情，图例可点击筛选，自适应窗口

## 四、个人阅读画像（新增核心部分）
基于已获取的历史数据，生成一段自然语言总结 + 辅助图表，内容包括：

### 4.1 阅读节奏分析
- 日均阅读时长是否稳定？波动大吗？
- 阅读天数占比（阅读天数 / 总天数）是多少？
- 若有多周数据，对比当前与上一周期的总时长变化趋势。

### 4.2 阅读偏好分析
- **最常读的作者**：统计阅读时长最长的前 3 位作者（完整姓名）
- **最常读的分类**：统计阅读时长最长的前 3 个分类
- **阅读媒介偏好**：电子书 vs 纸质书（若有数据）
- **平均每本书阅读周期**：总阅读天数 / 总书籍数（近似值）

### 4.3 阅读深度分析
- **笔记与划线的频率**：平均每本书的笔记数 = 总笔记数 / 总书籍数；划线数同理。
- **书籍完成度**：计算已读完（进度 ≥ 95%）的书籍数量及占比。
- **深度阅读指数** = （笔记数 + 划线数）/ 阅读时长（分钟）× 100，反映单位时长的思考密度。

### 4.4 阅读成长建议
基于以上分析，生成 3~5 条具体的、可执行的改进建议，例如：
- 建议增加某类书籍的阅读比例
- 建议每天固定时段阅读以提高连续性
- 建议对重点书籍多做笔记提升深度

### 4.5 画像可视化辅助
- 用雷达图（或柱状图）展示用户在“广度、深度、规律性、专注度”四个维度的得分（0~100 分，基于上述指标计算）
- 用词云（或表格）展示高频作者和分类

## 五、设计规范
- 配色：柔和渐变色（推荐 #667eea → #764ba2）
- 布局：顶部概览卡片 → 统计图表（两列布局）→ 阅读画像（文字 + 雷达图）→ 建议区域
- 所有中文标签，交互式（tooltip、图例筛选），响应式适配。

## 六、输出要求
- 生成一个独立的 HTML 文件，样式脚本内嵌（仅 ECharts 从 CDN 加载）
- 文件保存为 `weread_full_report_with_profile.html` 到当前目录
- 执行完成后输出：“报告已生成：weread_full_report_with_profile.html”

## 七、执行规则
- 用户可指定时间段（未指定则默认本周）
- 不要输出任何推理或思考过程，直接执行
- 若某部分数据不完整，用“暂无数据”展示，继续生成其余内容
