请利用 weread-skills 获取微信读书数据，生成一份包含阅读统计报告和个人阅读画像的交互式 HTML 可视化报告。

## 一、数据获取（完整提取，不截断）
请依次调用以下 API，并**完整提取**所有字段：

1. **`/readdata/detail?mode=weekly`**（默认本周，用户可指定 month/year）
   - 提取每本书：`title`、`author`（完整拼接）、`readingTime`（分钟）、`progress`（%）、`category`（分类）、`cover`（URL）
   - 同时提取：`readDays`（阅读天数）、`dailyReadTime`（每日时长数组）

2. **`/readdata/stat?mode=weekly`**
   - 提取：`totalReadingTime`、`totalNotes`、`totalBookmarks`

3. **`/readdata/stat?mode=year`**（获取年度累积数据）
   - 提取：`totalReadingTime`（年度总时长）、`totalBooks`（年度总书籍数）、`totalNotes`（年度总笔记）

4. **`/readdata/stat?mode=overall`**（获取历史总数据，如有）
   - 提取：`totalReadingTime`（历史总时长）、`totalBooks`（历史总书籍数）、`totalNotes`（历史总笔记）、`readDays`（历史总天数）、`authors`（去重作者数）、`categories`（去重分类数）

5. **对阅读时长最长的前 5 本书**调用 `/book/info`，提取：书名、作者、出版社、ISBN、分类、字数

6. **若用户要求对比**（如“与上周对比”），额外获取上一周期数据。

7. **缺失数据处理**：若某接口无数据，标记“暂无数据”，不中断生成。

## 二、字段展示规则（修复截断）
- 所有文本字段在 HTML 中**完整显示**，不得截断。
- 作者若为数组，用「、」连接所有作者。
- HTML 中设置 `white-space: normal; word-break: break-word;`，确保长文本换行。
- 缺失字段显示“暂无数据”。

顶部概览卡片：
- 总阅读时长、阅读天数、日均时长、总书籍数、总笔记数、总划线数

图表（ECharts 5，CDN 加载）：
1. 水平条形图：书籍阅读时长排行（显示完整书名和作者）
2. 环形图：各书籍阅读时长占比
3. 折线图（若有每日数据）：每日阅读趋势
4. 分类柱状图（若有分类数据）：常读分类分布
5. 所有图表支持悬浮、图例筛选、自适应

## 四、个人阅读画像（深度洞察版）

### 4.1 数据基石（自动计算）
基于 API 返回数据，自动计算以下关键指标：
- **阅读天数**：累计阅读天数（从历史总数据中提取）
- **日均阅读时长**：总时长 ÷ 总天数（分钟）
- **听书占比**：若 API 区分听书/阅读时长，计算占比；若无则标注“暂无区分”
- **总书籍数**：历史累计书籍总数
- **总笔记数**：历史累计笔记总数
- **平均每书笔记数**：总笔记数 ÷ 总书籍数
- **分类覆盖数**：去重分类数量
- **作者覆盖数**：去重作者数量
- **年度增长数据**：对比近两年年度总时长，计算增长率
- **完成书籍数**：进度 ≥ 95% 的书籍数量
- **热门书籍/作者**：按时长排序前列

### 4.2 阅读节奏分析
- 描述累计阅读天数（如“连续阅读 1148 天，相当于三年多从未间断”）
- 日均时长（如“日均 7 小时”）
- 若存在好友排名数据，引用排名信息（如“本周在好友中排名 #1”）
- 判断规律性：给予标签（如“近乎偏执的坚持”、“规律型”、“波动型”）
- **关键洞察**：结合听书占比，推测行为模式（如“96% 听书说明你是碎片时间大师”）

### 4.3 阅读深度分析
- 总笔记数 + 平均每书笔记数（如“1701 本书留下 49304 条笔记，平均每本 29 条”）
- 书籍完成数 + 完成率（如“已读完 262 本”）
- **关键洞察**：判断是否“被动接收”还是“持续与文本对话”
- **深度标签**：如“不止于‘听过’”、“深度对话型”

### 4.4 阅读广度分析
- 分类覆盖数 + 具体列出分类（如“横跨 8 大分类”）
- 作者覆盖数（如“涵盖 653 位作者”）
- **关键洞察**：判断是否“跨边界探索者”，是否兼顾中外著作
- **广度标签**：如“跨越边界的探索者”、“终身学习者”

### 4.5 阅读偏好分析
- **按分类排名**：列出时长最长的前 3 个分类及其具体分钟数（如“经济理财 2,791,491s”）
- **按作者排名**：列出时长最长的前 3 位作者（完整姓名）
- **时段偏好**：若有阅读时段数据，分析活跃时段（如“下午与夜晚读者”）
- **关键洞察**：结合分类偏好，分析阅读背后的动机（如“目标导向”、“追求认知升级”）

### 4.6 增长轨迹分析
- **若有多年度数据**：
  - 列出近两年年度总时长，计算变化率（如“2024 年 104h → 2025 年 2299h，增长 20 倍”）
  - 若当前年份数据可用，显示当前年份已累计时长及同比增速
- **关键洞察**：推测爆发原因（如“可能是有声书的采用”）
- **趋势判断**：“线性增长”还是“质变拐点”，给予标签（如“爆发式跃迁”）

### 4.7 个人综合评价
综合所有维度，用 **4~6 句话** 自然语言总结，格式参考：
> “你是一位「高密度输入型」阅读者，以近乎运动员般的自律，将阅读/听书变成了像呼吸一样自然的习惯。XX% 的听书比例让你成为……典型代表。你的阅读不是消遣，而是系统性的自我建设——[三大支柱分类]构成了一套完整的「认知升级工具箱」。从 [年份] 开始的爆发式增长，说明你已经找到了最适合自己的输入方式。保持这个节奏，你不仅是在读书，你是在用书重塑自己。”
